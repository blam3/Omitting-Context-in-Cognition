# Theory Loop

## Purpose

Translate the informal project thesis into formal objects and theorem candidates.

## Steps

1. PI Agent defines the target claim.
2. Mathematical Formalist defines notation, models, and assumptions.
3. Proof Critic searches for hidden assumptions.
4. Reviewer 2 Agent evaluates whether the formalization answers the scientific question.
5. New assumptions are routed to `docs/proposed_assumptions.md`.
6. Human approves or rejects new assumptions.
7. Approved theorem candidates move to the proof loop.

## Output files

- `docs/proposed_assumptions.md`
- `docs/assumption_registry.md`
- `docs/claim_registry.md`
- `manuscript/supplement_proofs.tex`
