# Reviewer Response Loop

## Purpose

Anticipate and answer methodological objections before submission.

## Steps

1. Reviewer 2 Agent generates objections.
2. PI Agent ranks objections by severity.
3. Simulation Architect proposes sensitivity checks.
4. Mathematical Formalist proposes theorem revisions if needed.
5. Manuscript Editor inserts limitations and clarifications.
6. Human approves final response strategy.

## Output files

- `docs/reviewer_objections.md`
- `docs/claim_registry.md`
- `manuscript/main.tex`
